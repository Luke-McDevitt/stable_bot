Demo: rolling ball

Generates a tilt-rotation routine so a ball on top orbits at the given radius. After generating, the routine auto-appears in the Play dropdown.

  • Ramp-up: spin-up phase. The orbit radius R grows 0 → R_final via a smoothstep. During this phase the tilt includes a tangential component (phase lead) that physically pushes the ball along the orbit, building up angular momentum. Shorter = more aggressive kick; longer = gentler.
  • Hold cycles: full orbit rotations at the steady-state tilt (R̈=Ṙ=0, just centripetal).
  • Ramp-down: mirror of ramp-up. R shrinks R_final → 0 with a braking phase lead that drains the ball's tangential velocity back to rest.
  • Center hold: flat platform (tilt = 0°) so the ball can come to rest on top before the routine ends.
  • Min tilt: safety floor during the active phases (not applied during center hold). Usually leave at 0 with the new feed-forward; only useful if you see the ball sitting still during early ramp-up because of rolling friction.

Set ramp-up or ramp-down to 0 to disable it (instant jump to/from the hold tilt — useful for breaking stiction but motors will slew hard).

Tip: at R=180 mm the min safe period under the 15°/s cap is ~3 s. First try without a ball (do not put a ball on it until you like what you see).

CAN recovery

If that fails → manual steps
  1. From Windows admin PowerShell:
    usbipd list usbipd detach --busid <X-Y> usbipd attach --wsl --busid <X-Y>
  2. If the adapter's LED stayed dark during detach/attach, physically unplug the USB cable and re-plug it, then re-run usbipd attach.
  3. In WSL: sudo ip link set can0 up type can bitrate 1000000
  4. Click "Soft-reset CAN bus" again.

To enable the soft-reset button (passwordless sudo for ip link):

sudo visudo -f /etc/sudoers.d/stewart-can
# add this line (replace 'sorak' if needed):
sorak ALL=(ALL) NOPASSWD: /usr/sbin/ip link set can0 *
